# Лаборатори 11 — Log4j2 болон Лог Шинжилгээ

Энэхүү төсөл нь Java програм дээр **Apache Log4j2** санг ашиглан лог хөтлөх, түүнийг тохируулах болон командын мөрнөөс (Terminal) лог файлыг шинжлэх үндсэн аргуудыг харуулсан болно.

## 1. Ангиудын тайлбар

* **BankAccount.java:** Дансны гүйлгээний (Орлого, Зарлага) логикийг агуулсан анги. Энэ ангид 6 өөр түвшний (TRACE, DEBUG, INFO, WARN, ERROR, FATAL) лог гаргах функцууд багтсан.
* **Customer.java:** Хэрэглэгчийн мэдээллийг хадгалах анги. Үүн дээр NullPointerException алдааг лог ашиглан хэрхэн илрүүлж, засах туршилтыг гүйцэтгэсэн.
* **App.java:** Програмын үндсэн оролт (Main class). Дээрх классуудыг дуудаж, туршилтын өгөгдлүүдээр ажиллуулах үүрэгтэй.

## 2. Log4j2 Тохиргоо (`log4j2.xml`)

Лог хөтлөлтийг дараах байдлаар тохируулсан:
* **Console Appender:** Консолд хугацаа, thread ID, түвшин, анги болон мөрийн дугаарыг хэвлэнэ.
* **RollingFile Appender:** Логуудыг `logs/app.log` файлд хадгална.
* **ROLL_BY_SIZE:** Файлын хэмжээ **1 MB** хүрэх бүрт шинэ файл үүсгэх (Rotate) болон өдөр бүр шинэ файл үүсгэх бодлогыг (`Policies`) нэмсэн.
* **Root Level:** Анхдагч лог түвшинг `DEBUG` болгож тохируулсан.

## 3. Алдааг логоор илрүүлж зассан түүх

`Customer` ангийн `getDomain()` метод дээр имэйл утга `null` байх үед `NullPointerException` үүсэж байсан.

**Илрүүлсэн явц:**
1. `logger.debug("getDomain called with email={}", email);` мөрийг нэмж имэйл утгыг хянасан.
2. Терминал дээр имэйл утга `null` орж ирж байгааг харж алдаа хаанаас үүсэж буйг олсон.

**Засвар:**
`if (email == null)` гэсэн шалгалтыг нэмж, `WARN` лог хэвлээд "UNKNOWN" гэсэн утга буцаадаг болгож зассан.

## 4. Даалгаврын асуултын хариулт

* **Даалгавар 2.1:** `app.log` файлыг шалгахад TRACE-аас FATAL хүртэлх бүх 6 түвшний лог амжилттай бүртгэгдсэн байна.
* **Даалгавар 4.1:** `tail -f` (Windows-д `Get-Content -Wait`) нь файлыг хаалгүйгээр шинээр нэмэгдэж буй логуудыг бодит цаг хугацаанд урсгаж харуулдаг. Энэ нь програм ажиллаж байх явцад алдааг шууд илрүүлэхэд тусалдаг бол `cat` нь зөвхөн тухайн агшин дахь мэдээллийг харуулдаг тул мониторинг хийхэд тохиромжгүй.

## 5. Шилдэг туршлага (Best Practices)

* Програмд String холболт (`+`) биш, параметрт формат (`{}`) ашиглан лог хэвлэсэн.
* Хэрэглэгчийн нууц мэдээллийг (Имэйл, ID) `mask()` функц ашиглан нууцалж логлосон.
